[b][size=15pt]A coin that died — and why it's back[/size][/b]

NADO started with a stubborn idea: that mining should be for [i]everyone[/i]. No graphics cards, no warehouses of ASICs, no needing to already be rich — just open a node and get a fair shot at every block. It borrowed the best instincts of NANO's distribution, IDENA's one-person-one-share ideal, and NYZO's cycle, and for a while it ran.

Then it died. Not to a hack — to its own design. Look closely at the original and the cracks are obvious:

[list]
[li]Rewards were handed out [b]by public IP address[/b] — one node per IP. Noble in theory. But an IP is the cheapest thing on the internet to rent: one cloud/VPS account spins up [i]hundreds[/i] of "unique" nodes across data centers for pocket change — while the real human on a phone behind carrier NAT got nothing (and needed a public IP just to play). The fairness ran backwards.[/li]
[li]Emission was [b]purely elastic[/b] — zero reward when the chain was quiet. So in the quiet stretches miners simply drifted away, and a network is only as alive as the people running it.[/li]
[li]A [b]"burn-to-bribe"[/b] knob let you pay to raise your own odds — which, however you frame it, is a lever that favors whoever has the most to spend.[/li]
[li]And the keys were [b]Ed25519[/b] — classical elliptic-curve crypto, already living on borrowed time against a quantum computer.[/li]
[/list]

The dream was right. The engine was wrong.

So we didn't patch it — we rebuilt it. Consensus, the mining model, the economics, the cryptography, and the client have been reworked almost end to end, then relaunched from a [b]fresh genesis with zero premine[/b] (chain id [tt]nado-relaunch-1[/tt]). If you knew the old NADO, this is not the same machine under the hood. The commit history tells the story better than we can — this was a ground-up overhaul.

[b]This is that reboot.[/b]

[b][size=15pt]Then → Now[/size][/b]

[list]
[li][b]Rewarded by IP[/b] (farmable from a datacenter)  →  [b]farm-neutral two-lane mining[/b]: a free lane hard-capped at ~20% of blocks, plus a stake lane priced in real, non-fakeable capital. Spinning up more cheap identities buys you nothing.[/li]
[li][b]Public IP required[/b] (excludes phones)  →  [b]phone-mineable in a plain browser tab[/b], behind NAT, screen locked.[/li]
[li][b]Zero reward when idle[/b]  →  a [b]base-subsidy floor[/b] so a brand-new, zero-coin miner earns from block one.[/li]
[li][b]Burn-to-bribe[/b] (pay to win)  →  a [b]presence dividend[/b] that streams most of every block's reward to everyone who shows up.[/li]
[li][b]Ed25519[/b] (quantum-vulnerable)  →  [b]ML-DSA-44[/b] (NIST FIPS 204 / Dilithium — post-quantum).[/li]
[li][b]"Stay online longer than the attacker"[/b]  →  [b]objective finality, equivocation slashing, and a Sybil ceiling proven by an adversarial test[/b] that floods the free lane with 100,000 fake identities and shows they still can't cross 20%.[/li]
[/list]

[b][size=15pt]How it works — two lanes, no race[/size][/b]

Block production is [b]not[/b] a hash race. One hash over a public randomness beacon deterministically picks each block's producer, so faster hardware confers no advantage and there is nothing to grind. Each epoch's slots split into two lanes:

[list]
[li][b]OPEN lane (~20%) — capital-free.[/b] Anyone wins with zero coins. Entry/upkeep is a tiny, [i]non-parallelizable[/i] sequential proof-of-work (~1 second on a phone) that grants a renewable ~1-day presence lease — cheap for a human, brutal for a GPU trying to mint millions of masks. This lane is [b]hard-capped at ~20% of blocks no matter how many identities exist[/b] — the population-independent Sybil ceiling that the old IP model never had.[/li]
[li][b]BONDED lane (~80%) — passive staking.[/b] Won in proportion to refundable, whale-capped, slashable stake. No work, no need to be online — you're credited by address and a relay builds your block. And a sudden whale can't seize the next epoch: a fresh bond's selection weight ramps up over ~30 epochs (while finality/quorum weighting stays untouched).[/li]
[/list]

[b]Everyone mining earns — a presence dividend, not a lottery.[/b] Winner-take-all blocks leave most miners with nothing for long stretches; that's what emptied the old network. Instead, NADO streams most of each block's reward to [b]everyone present[/b], weighted by how steadily they show up — a wage, not a jackpot. It accrues off-chain (a million miners cost the chain nothing, no dust bloat) and you sweep it to your balance with one Collect tap, claimed trust-minimised against a bonded-quorum-settled state root. Both lanes fund it.

[b][size=15pt]Fair launch — genuinely[/size][/b]

[list]
[li]Genesis mints [b]zero[/b] coins. No premine, no presale, no ICO, no insider allocation. Every coin was mined.[/li]
[li]Flat base subsidy of [b]0.1 NADO/block[/b] (rising with real fee activity to 0.5), so a zero-coin miner earns from block 1.[/li]
[li]Fees are [b]burned[/b], not paid to the producer. Emission is roughly constant in absolute terms, so the inflation rate trends toward zero over time.[/li]
[li]The only "allocation" is a 10% per-block treasury cut — and it starts empty, filling only from mined blocks.[/li]
[/list]

[b][size=15pt]Security[/size][/b]

[list]
[li][b]Post-quantum signatures[/b] — ML-DSA-44, cross-validated against the browser's implementation so a phone and a full node verify each other.[/li]
[li][b]Objective, grind-proof fork choice[/b] + enforced finality (a persisted floor rollback refuses to cross), FFG-lite stake-attested finality, commit-reveal RANDAO beacon, detached winner signatures with [b]equivocation slashing[/b].[/li]
[li][b]Anti-takeover by design.[/b] The exact failure that kills cycle/IP-based chains has no path here: the cheap lane is capped at 20% (machine-checked on every commit), and the rest costs real, farm-neutral capital a bot cannot acquire more cheaply than a human. It's documented, argued, and tested — not asserted.[/li]
[/list]

[b][size=15pt]Roadmap[/size][/b]

[list]
[li][b]zk-STARK shielded pool — post-quantum privacy.[/b] Optional, fully-shielded confidential transactions — hiding sender, recipient [i]and[/i] amount, with a pool-wide anonymity set — built on hash-based STARKs. Monero/Zcash-style elliptic-curve privacy is a non-starter on a post-quantum chain: a quantum computer breaks it, and for confidential amounts it could [i]silently inflate the supply[/i]. STARKs are post-quantum sound and need no trusted setup; the pool lives on the execution layer and reuses the same verifier planned for settlement. This is the flagship privacy direction — no elliptic-curve privacy will ever be bolted onto a chain built to survive quantum.[/li]
[li]Eclipse hardening (ASN-level peer diversity, pinned multi-seed bootstrap, checkpoint-bound snapshot sync).[/li]
[li]Execution-layer programmability (contracts) + a single validity-proof (STARK) settlement path.[/li]
[li]First-party mobile packaging on top of the browser miner.[/li]
[/list]

[b][size=15pt]How to mine[/size][/b]

[b]The easy way (phone or laptop):[/b] open any node's URL in a browser, tap [b]Generate key[/b], save it, tap [b]Start mining[/b]. Done — it registers and mines on its own, even with the screen locked. Nothing to install.

[b]Full node — Linux (direct):[/b]
[code]screen -S nado
sudo apt-get update
sudo apt-get install python3.10
python3.10 -m pip install --upgrade pip
git clone https://github.com/hclivess/nado
cd nado
python3.10 -m pip install -r requirements.txt
python3.10 nado.py[/code]

[b]Linux — one-liner:[/b]
[code]sudo apt-get update && sudo add-apt-repository ppa:deadsnakes/ppa -y && sudo apt-get install python3.10 python3.10-distutils git screen curl -y && curl -sS https://bootstrap.pypa.io/get-pip.py | sudo python3.10 && sudo ufw allow 9173 && git clone https://github.com/hclivess/nado && cd nado && python3.10 -m pip install -r requirements.txt && screen -dmS nado python3.10 nado.py[/code]

[b]Linux — virtualenv:[/b]
[code]sudo apt-get install python3.10-venv git -y
python3.10 -m venv nado_venv && source nado_venv/bin/activate
git clone https://github.com/hclivess/nado && cd nado
pip install --upgrade pip && pip install -r requirements.txt
python3.10 nado.py[/code]

[b]Windows:[/b] grab the [tt]nado.exe[/tt] binary from the [url=https://github.com/hclivess/nado/releases]releases page[/url], or install [url=https://www.python.org/downloads/]Python[/url], then [tt]python -m pip install -r requirements.txt[/tt] and [tt]python nado.py[/tt].

After it's running, open [tt]http://127.0.0.1:9173/[/tt] to mine from the page, and open/forward [b]port 9173[/b] so your node is reachable and earns. Reattach the screen session with [tt]screen -r nado[/tt]; update anytime with [tt]git pull origin main[/tt].
