🛡️ Shielded pool

A post-quantum zk-STARK shielded pool on the execution layer. Deposit here, withdraw to a different address: on-chain, your withdrawal is a zero-knowledge proof that hides the amount and which note you spent.
⚠ Private on-chain — but the prover sees your amounts, unless you run your own node. On-chain, your transfer is a real post-quantum zk-STARK, so amounts and the deposit→withdraw link are hidden from everyone reading the chain. For now the proof is generated by the exec node you're connected to (delegated proving), which sees your note's secrets while proving — so for full privacy, run your own exec node (or wait for on-device proving). Note: deposit and withdrawal amounts are public on L1 — real privacy comes from shielded transfers inside the pool (below) + a busy pool. Deposits/withdrawals are always secure. Settings

Defaults to this page's origin. A custom relay must be same-origin or send CORS headers, or your browser will block reads.

🔐 Wallet security

Encrypt the private key stored in this browser with a password (AES-256-GCM). It's decrypted only in memory while unlocked, and auto-lock re-locks it after a period of inactivity.

Self-test —

Proves this browser's canonical encoding, blake2b hashing, address derivation, registration PoW, ML-DSA-44 key derivation, txids and signatures are byte-identical to the node's Python (vectors generated from the live repo).

NADO light-miner · the chain stays on full nodes; this page only generates keys, registers, and heartbeats. Your reward lands on your address because a relay builds the block crediting it.
